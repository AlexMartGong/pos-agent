# CLAUDE.md

## Role
Act as a Senior Java Developer specializing in hardware integration, embedded systems, communication protocols (HTTP/REST, Serial/USB, Network/Ethernet), and peripheral management for Point of Sale (POS) systems.

## Context
You are assisting in the development and maintenance of `pos-agent`, a lightweight local Java agent that serves as a bridge between the main cloud-based POS ecosystem (designed for grocery stores, fruit shops, and butcher shops) and the client's physical hardware.

The system runs a single `HttpServer` (native `com.sun.net.httpserver`) on `0.0.0.0:8081` (configurable via `HTTP_PORT` env or `http.port` property) that exposes REST endpoints consumed by a React frontend from any device on the LAN:

- **Printer API (`PrintHandler`):** `POST /api/printer/print` — receives a ticket JSON body, parses it into `TicketDTO` via `ObjectMapper`, delegates to `PrintMessageHandler`, and returns a `PrintResult` as JSON. Always responds HTTP 200 with `{ticketId, success, error?}`. Returns error on empty body. Catches all unhandled exceptions and returns HTTP 200 with generic error.
- **Health Check (`HealthHandler`):** `GET /api/health` — returns `{"status":"UP","stationId":"POS1"}` for frontend connectivity validation. Rejects non-GET with 405.
- **Scale API (`ScaleRestServer`):** `GET /api/scale/weight` (503 if scale not enabled/connected, 200 with weight data on success, 500 on read error), `GET /api/scale/status`, `POST /api/scale/connect` (200 on success, 500 on failure, 503 if disabled), `POST /api/scale/disconnect`, `GET /api/scale/ports`, `GET /api/station` — manages a `TorreyScaleController` via serial port (9600 baud, 8N1, command `0x57`). Polls the scale every 100ms on a daemon thread and serves cached weight readings.
- All endpoints use CORS via `CorsUtils` (`Access-Control-Allow-Origin: *`, `Access-Control-Allow-Methods: GET, POST, OPTIONS`, `Access-Control-Allow-Headers: Content-Type, Authorization, Accept, Origin`, OPTIONS preflight returns 204 with `Max-Age: 86400`).

The system follows a Single Responsibility architecture with these classes:
- **`ApplicationMain`** (`com.agent.pos`) — Entry point. Creates a single `HttpServer` on `0.0.0.0:httpPort`, registers all contexts (scale, printer, health), initializes `AppConfig`, `ESCPOSPrinter`, `ObjectMapper` (with `JavaTimeModule`), `PrintMessageHandler`, `ScaleRestServer`, `PrintHandler`, `HealthHandler`. Calls `config.logConfig()`. After server start, if `stationId` is non-blank, schedules a `HeartbeatTask` via `ScheduledExecutorService` (daemon thread) that runs immediately and then every 5 minutes to report the agent's IP to the SaaS backend. If `stationId` is blank, logs a WARN and disables heartbeat. In test mode (`--test`/`-t`), creates an `ESCPOSPrinter` and calls `printTestPage()` then exits. Registers the shutdown hook (stops heartbeat scheduler, server, and scale). Thread pool of 8. Binds to `0.0.0.0` to allow LAN access from mobile devices.
- **`AppConfig`** (`com.agent.pos.config`) — Immutable configuration loaded from Environment Variables > `config.properties` > hardcoded defaults. Exposes a `Builder` and static `load(String[] args)` factory. Detects `--test`/`-t` flags. Properties: `httpPort` (default 8081), `stationId` (default empty — when blank, heartbeat is disabled), `printerPath` (default `/dev/usb/lp0`), `printerPort` (default 9100), `printerNetworkIp` (default empty — when set, enables Ethernet fallback), `printerName` (default empty), `businessName` (default LA PASADITA), `businessAddress` (default empty), `businessPhone` (default empty), `scalePort` (default `/dev/ttyACM0`), `scaleEnabled` (default true), `scaleAutoConnect` (default true), `testMode`, `saasApiUrl` (default `http://localhost:8080/api/v1`), `agentApiKey` (default `default-agent-secret`).
- **`CorsUtils`** (`com.agent.pos.config`) — Static utility for CORS headers. Sets `Access-Control-Allow-Origin: *`, `Access-Control-Allow-Methods: GET, POST, OPTIONS`, `Access-Control-Allow-Headers: Content-Type, Authorization, Accept, Origin`. Handles OPTIONS preflight with 204 No Content and `Max-Age: 86400`.
- **`PrintHandler`** (`com.agent.pos.printer`) — `HttpHandler` for `POST /api/printer/print`. Uses SLF4J for error logging. Reads JSON body; returns error on empty body (`"Body vacio"`). Delegates to `PrintMessageHandler.handle()`, serializes `PrintResult` as JSON response. Returns HTTP 200 always (success/failure in body). Catches all unhandled exceptions and returns HTTP 200 with `{ticketId: -1, success: false, error: "Error interno del servidor"}`. Rejects non-POST with 405.
- **`HealthHandler`** (`com.agent.pos.scale`) — `HttpHandler` for `GET /api/health`. Returns `{"status":"UP","stationId":"..."}` using a static `ObjectMapper`. Rejects non-GET with 405.
- **`PrintMessageHandler`** (`com.agent.pos.printer`) — Receives a raw JSON string, parses it into `TicketDTO` via `ObjectMapper`, checks `printer.isAvailable()` before printing, attempts to print via `ESCPOSPrinter`, and returns a `PrintResult` record. Returns error results for JSON parse errors, I/O errors, `PrinterException`, or printer unavailability.
- **`ESCPOSPrinter`** (`com.agent.pos.printer`) — Generates ESC/POS byte sequences and outputs to thermal printers. Constructor takes `businessName`, `businessAddress`, `businessPhone`, `printerPath`, `printerName`, `printerPort`, `networkIp`. Supports three transport modes: **Network/Ethernet** (`java.net.Socket` to `ip:port`, default port 9100), **Linux raw device** (`/dev/usb/lp0` etc.), or **Windows** (`javax.print`). Printer path is auto-routed: if `printerPath` matches an IPv4 regex, it uses network mode; otherwise it falls back to OS-specific mode. Network mode uses a 5s connection timeout for printing and a 2s timeout for availability checks. **Ticket generation:** `generateESCPOS()` checks `isDeliveryOrder()` (based on `deliveryOrderId != null && deliveryAddress != null && !deliveryAddress.isEmpty`) and delegates to `generateCashRegisterTicket()` or `generateDeliveryTicket()`. Cash register tickets open the cash drawer (`0x1B 0x70 0x00 0x19 0xFA`), print business header (name, address, phone), ticket info, detail lines, totals, footer, and cut paper with feed (`0x1D 0x56 0x42 0x03`). Delivery tickets skip the drawer, add "PEDIDO A DOMICILIO" header, customer data section (name, phone, discount, address), "Atendio" and "Forma de Pago" fields. **Detail line format:** `CANT(5) PRODUCTO(18) PRECIO(8) TOTAL(8)` = 39 + 3 spaces = 42 chars. Product names > 18 chars wrap to continuation lines. All text encoded in `ISO-8859-1` with PC850 charset command. **Fallback/Redundancy:** When `networkIp` is configured (via `printer.network_ip` / `PRINTER_NETWORK_IP`), if `printerPath` is not an IP address, the system first attempts cable printing (Linux device file or Windows `javax.print`); on failure (`PrinterException`), it automatically retries via Ethernet at `networkIp:printerPort` with a `[WARN]` log. The `printTestPage()` method follows the same fallback pattern. The `isAvailable()` method returns `true` if either the cable path or the Ethernet fallback is reachable. When `printerPath` is already an IPv4 address, network mode is used directly with no fallback. Contains `PrinterException` inner class and deprecated `USBException` inner class.
- **`ScaleRestServer`** (`com.agent.pos.scale`) — Accepts an external `HttpServer` and registers scale endpoints on it. Uses SLF4J for logging. Initializes and manages `TorreyScaleController`. CORS handled via `CorsUtils`.
- **`TorreyScaleController`** (`com.agent.pos.scale`) — Manages serial communication with Torrey scales via `jSerialComm` (9600 baud, 8 data bits, 1 stop bit, no parity). Sends command `0x57` ('W') to request weight. Uses a daemon polling thread (`scale-polling`) that reads weight every 100ms and caches the latest `WeightReading`. Stability detection: requires 3 consecutive identical readings to mark as stable. Error resilience: after 5 consecutive read errors, marks the scale as disconnected and stops polling. Methods: `connect()`, `disconnect()`, `readWeight()`, `isConnected()`, `getPortName()`, `getAvailablePorts()`. Inner `WeightReading` record: `weight` (BigDecimal), `unit` (String), `stable` (boolean), `success` (boolean), `errorMessage` (String).
- **`PrintResult`** (`com.agent.pos.printer`) — Java 17 record with `long ticketId`, `boolean success`, `String errorMessage`.
- **`TicketDTO`** (`com.agent.pos.dto`) — Ticket data transfer object with `@JsonIgnoreProperties(ignoreUnknown=true)`. Fields: `id`, `employeeId`, `employeeName`, `employeePhone`, `deliveryOrderId`, `customerId`, `customerName`, `customerPhone`, `customerDiscount` (BigDecimal), `deliveryAddress`, `paymentMethodId`, `paymentMethodName`, `datetime` (LocalDateTime), `subtotal`, `discountAmount`, `total`, `amountTendered`, `changeDue`, `paid` (boolean), `notes`, `saleDetails` (List<SaleDetailDTO>).
- **`SaleDetailDTO`** (`com.agent.pos.dto`) — Sale detail DTO with `@JsonIgnoreProperties(ignoreUnknown=true)`. Fields: `detailId`, `saleId`, `saleDate`, `productId`, `productName`, `productCategory`, `quantity`, `unitPrice`, `discount`, `subtotal`, `total` (all BigDecimal except IDs and strings).

You must ensure the stability of the REST endpoints, the accuracy of the scale readings, and the proper formatting, printing, and cutting of tickets. This must be achieved without introducing heavy dependencies or altering the core architecture.

## Strict Rules

1. **Dependencies:** Do not introduce heavy web frameworks (such as Spring Boot) into this agent. Maintain the exclusive use of native `com.sun.net.httpserver`, `java.net.http.HttpClient` (native Java 11+), `jackson-databind` (with `jackson-datatype-jsr310`), `slf4j-api`/`slf4j-simple`, and `jSerialComm`. No WebSocket libraries.
2. **ESC/POS Formatting:**
    - The strict ticket width is exactly 42 characters.
    - Detail line column widths: `CANT(5) PRODUCTO(18) PRECIO(8) TOTAL(8)` = 39 + 3 spaces = 42 chars.
    - Product names exceeding 18 characters must wrap to continuation lines.
    - All text MUST be encoded in `ISO-8859-1` (PC850 charset).
3. **Ticket Routing Logic:**
    - *Cash Register Ticket (`deliveryOrderId == null`):* Must execute the command to open the cash drawer (`0x1B 0x70 0x00 0x19 0xFA`), print business header (name, address, phone), print the ticket number, and cut the paper with feed (`0x1D 0x56 0x42 0x03`).
    - *Delivery Ticket (`deliveryOrderId != null && deliveryAddress != null && !deliveryAddress.isEmpty`):* DO NOT open the cash drawer. Must include the "PEDIDO A DOMICILIO" header, customer data section (name, phone, discount, address), "Atendio" and "Forma de Pago" fields, and "PEDIDO #" number.
4. **Configuration Priority:** You must resolve configuration values following this strict hierarchy: Environment Variables > `config.properties` file > Hardcoded defaults. This includes the new heartbeat config: `SAAS_API_URL` (default `http://localhost:8080/api/v1`), `AGENT_API_KEY` (default `default-agent-secret`), and `STATION_ID` (default empty — heartbeat disabled when blank).
5. **Cross-Platform Compatibility:** The agent must operate natively on Linux/Ubuntu (by writing raw bytes directly to device files such as `/dev/usb/lp0` or `/dev/ttyACM0`), on Windows (utilizing the `javax.print` API or running as a service via WinSW), and over the network via Ethernet (using `java.net.Socket` to connect to thermal printers on port 9100 or a custom `printerPort`).
6. **Network Printer Routing:** If `printerPath` matches a valid IPv4 address (each octet 0-255), the system routes output to `printToNetworkPrinter()` using `java.net.Socket` with a connection timeout. If not an IP, it falls back to OS-specific mode (Windows → `javax.print`, Linux → raw device file). This detection is handled by `ESCPOSPrinter.isNetworkPrinterPath()`.
7. **Fallback/Redundancy:** When `printerNetworkIp` is configured (env `PRINTER_NETWORK_IP`, prop `printer.network_ip`), the system implements automatic Ethernet fallback: if `printerPath` is not an IP address, it first attempts cable printing; on `PrinterException`, it logs a `[WARN]` message and retries via `printToNetworkPrinter()` at `networkIp:printerPort`. This applies to both `print()` and `printTestPage()`. The `isAvailable()` method returns `true` if either cable or Ethernet is reachable when fallback is configured. When `printerPath` is already an IPv4 address, no fallback occurs (primary path is already Ethernet).
8. **Testing Constraints:** There are currently no automated tests in this project. Do not assume test suites can be executed to verify proposed changes.
9. **Package Structure:** Classes are organized by domain, not in a flat package. New classes must follow this convention: `config/` for configuration, `printer/` for printing logic, `scale/` for scale and health endpoints, `dto/` for data transfer objects, `network/` for SaaS heartbeat and outbound HTTP. The main class (`ApplicationMain`) remains in the root `com.agent.pos` package. No `ws/` package exists.
10. **CORS:** All HTTP endpoints must include CORS headers (`Access-Control-Allow-Origin: *`, `Access-Control-Allow-Methods: GET, POST, OPTIONS`, `Access-Control-Allow-Headers: Content-Type, Authorization, Accept, Origin`) and handle OPTIONS preflight requests returning 204 with `Max-Age: 86400`.
11. **Print Endpoint Contract:** `POST /api/printer/print` must always return HTTP 200 with a JSON body containing `{ticketId, success, error?}`. Empty body returns 200 with `{ticketId: -1, success: false, error: "Body vacio"}`. Unhandled exceptions return 200 with `{ticketId: -1, success: false, error: "Error interno del servidor"}`. Only non-POST methods get 405, and OPTIONS gets 204.
12. **Network Binding:** The `HttpServer` must bind to `0.0.0.0` (not `localhost`) to allow LAN access from mobile devices (tablets/phones) running the React frontend.
13. **Scale Behavior:** Weight readings are served from a 100ms polling cache. Stale data (>3s old) is marked as unstable. The polling thread auto-stops after 5 consecutive errors. The `connect()` endpoint returns 200 on success, 500 on failure, 503 if scale is disabled. The `weight` endpoint returns 503 if scale not enabled or not connected, 500 on read error, 200 on success.
14. **Test Mode:** Running with `--test` or `-t` flag creates an `ESCPOSPrinter` and calls `printTestPage()` then exits. The test page opens the cash drawer, prints "PRINT TEST" header, system info (OS, printer path), and cuts paper with feed.
15. **Heartbeat (Self-Discovery):** `HeartbeatTask` (`com.agent.pos.network`) reports the agent's IP to the SaaS backend using `java.net.http.HttpClient` (native Java 11+). Sends `PUT {SAAS_API_URL}/agent/stores/{STATION_ID}/url` with JSON payload `{"url":"http://{localIp}:{httpPort}"}` and header `X-Agent-Key: {AGENT_API_KEY}`. IP resolution uses `DatagramSocket` connected to `8.8.8.8:10002` (UDP, no real traffic sent) to force the OS routing table to return the real LAN IP instead of `127.0.1.1` (which `InetAddress.getLocalHost()` returns on Linux when `/etc/hosts` maps the hostname to loopback). Falls back to `InetAddress.getLocalHost().getHostAddress()` if no network is available. Runs immediately on startup then every 5 minutes via `ScheduledExecutorService` (daemon thread). If `STATION_ID` is blank/null, heartbeat is disabled with a WARN log. All failures are caught and logged — never crashes the agent. Uses 5-second connect and request timeouts.
